# MiniMesh — Development Phases

## Phase 0 — Repo Foundation
Clean project skeleton, documentation, environment setup, and runnable starter app.
FastAPI backend with health routes. React + Vite + Tailwind frontend with pipeline UI shell.

## Phase 1 — Upload Reference Images
File upload UI with drag-and-drop. Backend multipart upload endpoint. Local storage of uploaded references. Image preview and management.

## Phase 2 — Reference Sorting ✓ Complete
Backend metadata editing: reference_role, notes, is_primary per image. Project session with persistent name and generation readiness scoring (0–100). Frontend filter tabs, notes input, primary toggle, and ProjectSessionPanel showing score, status, missing/warnings/strengths.

## Phase 3 — Sculpt/Model Type Selector ✓ Complete
Eight output mode cards (2D Anime Sheet, 3D Model, Clay Sculpt, Toy/Figurine, Game-Ready Character, Cinematic High-Poly, Low-Poly Mobile, Prop Only). Generation config (style, rig intent, quality, texture style, notes) persisted to storage/generation_config.json. Mode-specific readiness requirements surfaced in ProjectSessionPanel.

## Phase 4 — Image-to-3D Provider Integration ✓ Complete
Job model (queued/processing/completed/failed) with provider abstraction layer (BaseProvider + MockProvider). Job service with readiness validation (rejects if not_ready, missing required mode refs, or no images). Routes: POST /api/jobs/generate, GET /api/jobs/{id}, GET /api/jobs. Mock provider simulates queued→processing→completed over ~12 seconds and writes result.json to exports/jobs/{job_id}/. Frontend: Generate Draft Mesh button with 2-second polling, inline job status in ConfigPanel, JobHistoryPanel showing all past jobs. UI clearly labels mock provider active.

## Phase 5 — Browser 3D Preview ✓ Complete
React Three Fiber (R3F v8) + @react-three/drei viewer foundation. Full-screen 3D viewport with OrbitControls, ambient/directional/point lighting with cyan+violet accents. ViewerToolbar: Reset Camera, Solid/Wireframe/Toon material modes, Auto Rotate, Grid toggle. ViewerInspector: job info, preview stats, rig/animation/texture/loader placeholders. ViewerStatusBar: live status strip. PlaceholderMesh: stylized geometric humanoid with slow idle rotation and materialMode-driven rendering. Viewer3D page opened from SculptTypeSelector "Open 3D Preview" button after job completes. Architecture ready for future GLB/GLTF/OBJ/FBX loaders.

## Phase 6 — Export System ✓ Complete
Export pipeline foundation with manifest system and ZIP packaging. Routes: POST /api/exports/{job_id}/create, GET /api/exports/job/{job_id}, GET /api/exports/{export_id}, GET /api/exports/{export_id}/download. Only completed jobs can export. Each package creates README.txt + manifest.json, zipped to bundle.zip under exports/packages/{export_id}/. An exports_index.json per job tracks all export IDs. ExportPanel in Viewer3D shows export list, file size, creation time, and ZIP download button. Architecture ready for real GLB/FBX/OBJ outputs in future phases.

## Phase 7 — Auto-Rigging Foundation ✓ Complete
Five built-in rig profiles (Humanoid, Creature, Winged Humanoid, Prop, Vehicle). RigJob model (queued/processing/completed/failed) with MockRigProvider (~8s simulation) writing rig_data.json to exports/rigs/{rig_job_id}/. Routes: GET /api/rigs/profiles, POST /api/rigs/create, GET /api/rigs/{id}, GET /api/rigs. RigStudio full-screen page: left panel with profile selector + rig history, center 3D viewport with SkeletonOverlay (cyan line segments + white node spheres), right RigInspector panel with bone list and metadata. Viewer3D: "Open Rig Studio →" button in header after job completes, Skeleton toggle in toolbar. Skeleton overlay visible in both Viewer3D and RigStudio. Mock provider clearly labeled throughout.

## Phase 8 — Animation Preview Foundation ✓ Complete
Seven built-in mock clips (Idle Pulse, Walk Cycle, Run Cycle, Hero Landing, Weapon Slash, Wing Hover, Turntable Preview). AnimationJob model (queued/processing/completed/failed) with MockAnimationProvider (~7s simulation). Routes: GET /api/animations/clips, POST /api/animations/create, GET /api/animations/{id}, GET /api/animations. AnimationPreview full-screen page: left panel with AnimationClipSelector (rig-type filtered) + AnimationJobPanel, center 3D viewport with MockMotionOverlay (per-clip-type bone animations via useFrame), right AnimationInspector, bottom AnimationTimeline (play/pause, restart, progress bar, 0.5x–2x speed, loop toggle). RigStudio: "Open Animation Preview →" button after rig completes. Home pipeline "Animate Preview" card active. Mock motion clearly labeled. Real retargeting/skinning/deformation reserved for future phases.

## Phase 9 — Advanced Rig Modules + Attachment System ✓ Complete
Six built-in rig modules (Wing, Shield, Weapon, Tail, Back Armor, Transform:Wings→Shield). Seven default attachment sockets (spine_back, left_hand, right_hand, waist, head, left_wing, right_wing) with position/rotation metadata. Four transform states (Flight, Defense, Combat, Idle). Module assignment validates rig_type compatibility and bone availability. Routes: GET /api/modules, GET /api/modules/sockets, POST /api/modules/assign, GET /api/modules/rig/{rig_job_id}. RigStudio: tabbed left panel (Profiles|Modules), RigModulePanel with multi-select and compatibility display, SocketVisualizer R3F component with glowing colored spheres + Html labels + connection lines to parent bones, ⊙ Sockets viewport toggle, TransformStatePanel in right panel with state switcher and active module highlights, socket color legend overlay. Metadata only — real IK, mesh transforms, and procedural animation reserved for future phases.

## Phase 10 — Quality Checker ✓ Complete
Project audit engine with 6-category pipeline scoring (References, Generation, Rigging, Animation, Exports, Modules). Score 0–100 with status: production_ready (90+), needs_attention (65+), unhealthy (<65). AuditIssue model with severity (critical/warning/info) and repair suggestions. Routes: POST /api/audits/run, GET /api/audits/latest, GET /api/audits. QualityDashboard full-screen page: Run Audit button, SVG circular score ring, pipeline health summary grid, grouped issue list (expandable rows), strengths list, prioritized recommendations, audit history panel. Entire audit stored as JSON in storage/audits/. All pipeline checks wrapped in try/except for robustness.

## Phase 11 — User Library ✓ Complete
Multi-project architecture with active project context routing. All services (uploads, jobs, rigs, animations, modules, exports, audits, generation config) now read/write through the active project's directory. Legacy project (id="legacy") maps to old flat storage layout — existing data preserved with no migration required. New projects get isolated storage under storage/projects/{id}/. Routes: GET /api/library/projects, POST /api/library/projects/create, POST /api/library/projects/{id}/duplicate, DELETE /api/library/projects/{id}, POST /api/library/projects/{id}/activate, GET /api/library/projects/{id}. Thumbnails derived from primary front-view image. Frontend: ProjectManager full-screen page with project grid, ProjectCard with thumbnail/score/mode/actions, CreateProjectModal with 4 templates, active project indicator in home header with quick-switch button.

## Phase 12 — Material Studio Foundation ✓ Complete
5 built-in shader presets (Anime Toon, Black Gold Armor, Holographic Energy, Matte Clay, Sci-Fi PBR). MaterialProfile model with shader_type (toon/pbr/metallic/holographic/matte/emissive), colors, metallic, roughness, emissive intensity, opacity, rim_light, toon_steps. Routes: GET /api/materials, POST /api/materials/create, PATCH /api/materials/{id}, DELETE /api/materials/{id}, POST /api/materials/{id}/activate. Active material stored per-project. MaterialStudio full-screen page: left preset browser, center R3F live viewer, right MaterialInspector (sliders, color pickers, shader type selector, rim light toggle), MaterialToolbar (Reset/Save/Duplicate/Randomize), ActiveMaterialBar with Apply to Project. PlaceholderMesh + MeshViewer extended to accept MaterialProfile prop for real-time shader preview. All 9 pipeline cards now active. Export Asset card opens Viewer3D export panel.

## Phase 13 — Credits + Usage Ledger ✓ Complete
Usage-based credit system. CreditWallet, LedgerEntry, PricingConfig models. Global wallet at storage/credits/wallet.json. Ledger entries for all 5 pipeline actions (generation=25, rigging=10, animation=8, export=5, material=2). Routes: GET /api/credits/wallet, GET /api/credits/ledger, GET /api/credits/pricing, POST /api/credits/mock-add. HTTP 402 raised on insufficient balance. InsufficientCreditsModal on frontend with +100/+500/+1000 mock top-up. CreditBalance HUD in home header (color-coded). CreditDashboard page with WalletSummary, Mock Checkout, PricingPanel, LedgerPanel. CreditContext threaded to all spending components. Cost badges on all action buttons. Balance refreshes instantly after every spend or top-up.

## Phase 14 — Real Provider Integration Foundation ✓ Complete
Provider abstraction upgraded with standardized submit_generation/poll_generation/download_result/normalize_result methods. Meshy image-to-3D integration: submits primary upload as base64 data URI to POST /openapi/v2/image-to-3d, polls status, downloads GLB to exports/jobs/{job_id}/model.glb, saves provider_result.json. Job model extended with external_job_id, preview_url, model_url, model_downloaded, glb_path, progress. Provider selection: if MESHY_API_KEY set → MeshyProvider, else → MockProvider. Existing jobs polled by their original provider. Routes: GET /api/providers/status, GET /api/providers/active. Frontend: RealModelViewer component (useGLTF + Suspense), MeshViewer conditionally loads real GLB, Viewer3D shows REAL MODEL/MOCK PREVIEW badge, ProviderStatusPanel shows capability/key status. Home header shows "Meshy Connected" or "Mock Provider Active" global badge. ViewerInspector shows external task ID, progress %, provider output section. ViewerStatusBar updated. httpx added to requirements. MESHY_API_KEY/TRIPO_API_KEY/RODIN_API_KEY env vars. Graceful failure: Meshy errors stored in job.error, never crash the app.

## Phase 15 — Provider Settings + Multi-Provider Expansion ✓ Complete
Provider registry (mock, meshy, tripo stub, rodin stub) with per-provider metadata, capabilities, priority order, enabled state, and health cache. provider_registry.py as single source of truth. provider_health_service.py with live ping for Meshy (latency, 401/429/timeout handling), instant responses for mock/stubs. Fallback chain in job_service: walks priority list on submit, logs each attempt as {name}_failed or {name}_submitted, hard-falls to mock if all fail. provider_attempts field on Job. Routes added: GET /api/providers/priority, PATCH /api/providers/priority, PATCH /api/providers/{name}/enabled, POST /api/providers/test/{name}. /api/providers/active now returns health_status + fallback_provider. Stub providers: TripoProvider, RodinProvider (NotImplementedError guards). Frontend: ProviderSettings page with summary strip, ProviderCard (capabilities, key badge, health badge, test button, enable/disable), ProviderPriorityEditor (up/down reorder, save), ProviderHealthBadge. Home header badge clickable → Provider Settings, shows health color + fallback label. JobHistoryPanel shows attempt chain for fallback jobs. Pipeline card 11 added.

## Phase 16 — Production Hardening
Full test coverage. Rate limiting and abuse prevention. CDN-backed storage. Horizontal scaling for worker jobs. Railway production deployment.
