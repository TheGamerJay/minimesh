# MiniMesh — Development Phases

## Phase 0 — Repo Foundation
Clean project skeleton, documentation, environment setup, and runnable starter app.
FastAPI backend with health routes. React + Vite + Tailwind frontend with pipeline UI shell.

## Phase 1 — Upload Reference Images
File upload UI with drag-and-drop. Backend multipart upload endpoint. Local storage of uploaded references. Image preview and management.

## Phase 2 — Reference Sorting ✓ Complete
Backend metadata editing: reference_role, notes, is_primary per image. Project session with persistent name and generation readiness scoring (0–100). Frontend filter tabs, notes input, primary toggle, and ProjectSessionPanel showing score, status, missing/warnings/strengths.

## Phase 3 — Sculpt/Model Type Selector ✓ Complete
Eight output mode cards (2D Anime Sheet, 3D Model, Clay Sculpt, Toy/Figurine, Game-Ready Character, Cinematic High-Poly, Low-Poly Mobile, Prop Only). Generation config (style, rig intent, quality, texture style, notes) persisted to storage/generation_config.json. Mode-specific readiness requirements surfaced in ProjectSessionPanel.

## Phase 4 — Image-to-3D Provider Integration ✓ Complete
Job model (queued/processing/completed/failed) with provider abstraction layer (BaseProvider + MockProvider). Job service with readiness validation (rejects if not_ready, missing required mode refs, or no images). Routes: POST /api/jobs/generate, GET /api/jobs/{id}, GET /api/jobs. Mock provider simulates queued→processing→completed over ~12 seconds and writes result.json to exports/jobs/{job_id}/. Frontend: Generate Draft Mesh button with 2-second polling, inline job status in ConfigPanel, JobHistoryPanel showing all past jobs. UI clearly labels mock provider active.

## Phase 5 — Browser 3D Preview ✓ Complete
React Three Fiber (R3F v8) + @react-three/drei viewer foundation. Full-screen 3D viewport with OrbitControls, ambient/directional/point lighting with cyan+violet accents. ViewerToolbar: Reset Camera, Solid/Wireframe/Toon material modes, Auto Rotate, Grid toggle. ViewerInspector: job info, preview stats, rig/animation/texture/loader placeholders. ViewerStatusBar: live status strip. PlaceholderMesh: stylized geometric humanoid with slow idle rotation and materialMode-driven rendering. Viewer3D page opened from SculptTypeSelector "Open 3D Preview" button after job completes. Architecture ready for future GLB/GLTF/OBJ/FBX loaders.

## Phase 6 — Export System ✓ Complete
Export pipeline foundation with manifest system and ZIP packaging. Routes: POST /api/exports/{job_id}/create, GET /api/exports/job/{job_id}, GET /api/exports/{export_id}, GET /api/exports/{export_id}/download. Only completed jobs can export. Each package creates README.txt + manifest.json, zipped to bundle.zip under exports/packages/{export_id}/. An exports_index.json per job tracks all export IDs. ExportPanel in Viewer3D shows export list, file size, creation time, and ZIP download button. Architecture ready for real GLB/FBX/OBJ outputs in future phases.

RigStudio full-screen page: left panel with profile selector + rig history, center 3D viewport with SkeletonOverlay (cyan line segments + white node spheres), right RigInspector panel with bone list and metadata. Viewer3D: "Open Rig Studio →" button in header after job completes, Skeleton toggle in toolbar. Skeleton overlay visible in both Viewer3D and RigStudio. Mock provider clearly labeled throughout.
